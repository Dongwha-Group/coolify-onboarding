# 프롬프트 가이드 룰: Commit 작성 방법

## 1. Commit 포맷

- 한 줄은 **100자(한글 50자) 이하**로 작성합니다.

### Commit 구조

```
<타입>: <제목>

<본문>

Related to: #Git Issue ID
```

## 2. `<타입>` 종류

- **feat**: 기능 추가
- **docs**: 문서 수정 (예: README.md, Swagger 어노테이션)
- **build**: 빌드 시스템 또는 종속성 변경
- **refactor**: 코드 리팩터링 (동작에 영향 없는 변경: 띄어쓰기, 세미콜론, 주석 등)
- **test**: 테스트 케이스 추가/수정
- **chore**: 간단한 소스 변경, UI/CSS 수정, 파일/폴더명 변경
- **fix**: 버그 수정

## 3. `<제목>` 작성 방법

1. **명령문**, **현재형**, **동사원형** 사용
   - 예: `change`, `add`, `fix`
2. 대문자로 시작하지 않기
3. 문장 끝에 마침표(`.`) 붙이지 않기

## 4. `<본문>` 작성 방법

1. **명령문**, **현재형**, **동사원형** 사용
2. `-` 기호를 사용하여 항목 구분
3. **변경 이유와 이전 대비 차이**를 상세히 작성
4. **무엇을 고쳤고, 왜 고쳤는지** 설명

## 5. 작성 예시

### 예시 1: 기능 추가

```
feat: 지난 날짜에 대한 송출 데이터 생성 배치 로직 개발

- 실행되지 않은 지난 날짜의 데이터를 생성하는 배치 로직 생성
- Local에서만 수행하며 Stage, Prod에는 스케줄 시간 주석 처리

Related to: #TEPR-6
```

### 예시 2: 문서 수정

```
docs: README.md 로컬 설치방법 추가

Related to: #TEPR-20
```

### 예시 3: 버그 수정/기타

```
chore: 미디어 전체 다운로드 엑셀에 부산지하철 시트 추가

- 미디어 전체 다운로드 엑셀 파일에 부산지하철 시트를 추가
- bsub25 타입의 매체들을 해당 시트에 출력되도록 수정

Related to: #TEPR-10
```
